# 用户画像与工作流痛点

## AI 产品经理

- 工作目标：把业务要求转成可验收的 AI 行为，并决定版本能否上线。
- 当前方式：在 PRD、表格和聊天窗口之间同步 Prompt 与样例。
- 核心痛点：需求与 Prompt 版本脱节；缺少统一指标；评测结论难复现。
- 使用场景：定义变量和验收 Case，查看版本评测，定位失败，发起改进。
- 成功标准：能用同一批 Case 解释版本质量、风险、成本和发布建议。

## Prompt Engineer / AI 应用开发者

- 工作目标：稳定迭代 Prompt 与模型参数，并快速定位调用问题。
- 当前方式：Prompt 硬编码或散落在配置文件，靠单次请求人工试错。
- 核心痛点：修改缺少快照；输入不固定；provider failure 与质量失败混在一起。
- 使用场景：编辑模板和变量、保存不可变版本、Playground 调试、查看 Invocation。
- 成功标准：任一结果都能回溯到 PromptVersion、输入、模型身份和原始输出。

## AI 测试 / 运营人员

- 工作目标：在上线前批量覆盖正常、边界和错误场景，持续沉淀回归集。
- 当前方式：表格维护 Case，逐条复制输入、记录输出并人工判定。
- 核心痛点：执行效率低、规则不统一、失败难重跑、历史容易被覆盖。
- 使用场景：维护 Dataset、配置规则、批量运行、筛选失败、Retry Case。
- 成功标准：批次状态清晰；skipped、调用失败和规则失败分离；历史记录保留。

## 跨角色共性痛点

Prompt 管理混乱、修改缺少版本依据、上线前缺少系统评测、测试结果不可复现、模型失败难定位、成本与性能不可观测，以及协作缺少统一质量门槛。当前 MVP 解决单机闭环；多人审批和发布门禁仍属于 Roadmap。
